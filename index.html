The Problem

Governments approve policies
without knowing their cost

Policy decision-making in Saudi Arabia — and across the GCC — lacks structured, data-driven tools for workforce impact analysis. The result is costly mismatches discovered only after implementation.

⚡

No Simulation Before Approval

Policies are evaluated based on static reports, not dynamic simulations of workforce demand, training capacity, or budget stress.

📊

Workforce Mismatches

Saudization targets, training pipeline gaps, and sector-specific labor shortages are discovered after projects have begun — not before.

🔍

No Alternative Exploration

Decision-makers receive one option — approve or reject. No tool exists to rapidly compare implementation strategies or policy variants.

⚠️

Portfolio Blind Spots

Multiple active policies are analyzed in isolation. Cross-policy conflicts — two initiatives competing for the same nurses, engineers, or training institutions — remain invisible until it's too late.

The Solution

AI that reasons about policy,
not just reports on it

AgentNation is a multi-agent decision support system where specialized AI agents collaborate in sequence — each responsible for a distinct stage of policy evaluation. The system surfaces risks, quantifies them, and generates recommendations. Every output is non-binding. Humans remain in control.

Use Cases

Three levels of intelligence

Micro → Meso → Macro. Each use case answers a different question at a different scale.

UC — 01 Micro

Policy Workforce Impact Assessment

A policymaker submits a proposed workforce policy. The system evaluates feasibility through a 5-agent pipeline and routes to one of three decision paths: No Risk, Monitor, or Mitigation Required.

PAA WIA MAA COA ORA
UC — 02 Meso

Policy Scenario Comparison

Instead of one policy, the user submits a policy intent and growth target. The SGA automatically generates three strategic scenarios. The CAA ranks them. A hybrid synthesis is available on demand.

SGA PAA WIA MAA CAA ORA
UC — 03 Macro

Workforce Portfolio Risk Monitor

A portfolio of 3–10 active policies is analyzed simultaneously. The PRA uses the Knowledge Graph to detect cross-policy conflicts — workforce overlap, capacity overload, and training bottlenecks.

WIA × N MAA × N PRA ORA
Agent Architecture

Every agent has one job

Sequential, transparent, and auditable. Each agent passes structured output to the next. No black boxes. Every decision is explainable.

PAA
Policy Analysis Agent
Reads policy text via Gemini LLM + RAG context. Extracts objectives, affected domains, assumptions, and complexity level.
→
WIA
Workforce Intelligence Agent
Reads sector database. Computes workforce pressure, budget stress ratio, training capacity gap, and Saudization risk.
→
MAA
Monitoring & Assessment Agent
Applies weighted scoring formula to WIA metrics. Outputs severity score 0–100 and risk classification.
→
COA
Coordination & Oversight Agent
Routes by numeric threshold to Path A (no risk), B (monitor), or C (mitigation). Triggers ORA if needed.
→
ORA
Option Recommendation Agent
Generates targeted policy alternatives or hybrid scenarios. Each recommendation addresses a specific detected risk flag.
⬡
UC3 Only — Portfolio Risk Agent

PRA — Portfolio Risk Agent

The PRA is the only agent that uses the Knowledge Graph. It aggregates all WIA and MAA outputs across the portfolio and queries the graph to detect cross-policy conflicts: Workforce Overlap, Capacity Overload, Training Bottleneck, and Budget Concentration. It computes a portfolio risk score and sector pressure map that are invisible when analyzing policies individually.

Intelligence Layer

Grounded in real Saudi data

Two intelligence systems make AgentNation's output specific, not generic.

🔍
RAG — Retrieval-Augmented Generation

44 curated Saudi policy documents

Before each LLM call, the agent retrieves the 4–5 most relevant passages from a document corpus embedded with Google text-embedding-004 and stored in ChromaDB. Documents include Vision 2030 Annual Reports, HSTP, GASTAT labor statistics, IMF Article IV, KAS Saudization reports, OECD benchmarks, Singapore SkillsFuture, and ILO Arab States Employment Outlook.

Without RAG: "Assume training pipeline is sufficient."
With RAG: "Healthcare training pipeline produces 8,000 nurses/year — policy requires 3.2× this output, consistent with HSTP 2025 projections."

⬡
Knowledge Graph — NetworkX

Sector relationship reasoning

A purpose-built directed graph with four relationship layers: Policy → Sector → Workforce Type → Training Institution, with HAS_CAPACITY edges storing workforce supply data. The PRA queries this graph to detect which policies share the same workforce pools or compete for the same training institutions.

Without KG: Healthcare expansion looks fine in isolation.
With KG: Two healthcare policies are competing for the same medical college graduates — 14,300 per year vs 32,000 needed.

Design Principles

Why it's built this way

01

Human Authority Preserved

No agent makes an autonomous decision. All outputs are explicitly labeled non-binding. ORA in UC2 only runs when the user clicks a button. The system surfaces risks — humans decide actions.

02

Deterministic Core

WIA, MAA, and COA are pure logic — no LLM calls. The risk score formula is published and auditable. LLM failures fall back gracefully without breaking the pipeline.

03

System Restraint

Path A skips MAA entirely when WIA metrics are clean. The system does not escalate without cause. This is as important academically as Path C — it proves the system is not an alarmist.

04

80% Agent Reuse

UC2 imports WIA and MAA directly from UC1. UC3 does the same. No code duplication. This demonstrates modular, scalable architecture — new use cases cost 20% of the original effort.

05

Grounded in Saudi Context

Every RAG document was selected for Saudi relevance — Vision 2030 milestones, GASTAT labor data, HSTP targets, KAS Saudization reports. The system reasons about the right context, not generic assumptions.

06

Transparent by Design

Every agent card in the dashboard expands to show its raw JSON output. The reasoning chain from policy text to final recommendation is fully visible and auditable at every step.
